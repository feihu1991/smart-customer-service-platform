## 测试覆盖详情

### 1. 用户登录 (auth.test.ts)
- 用户注册（新用户、重复注册、手机号验证）
- 密码登录（成功、失败、不存在用户）
- 验证码登录（成功、失败、无凭证）
- 获取用户信息（已登录、未登录、无效Token）
- 用户登出

### 2. 评价查看 (reviews.test.ts)
- 获取评价列表
- 分页查询
- 平台筛选（淘宝/天猫/京东/拼多多）
- 情感筛选（正面/负面/中性）
- 回复状态筛选
- 单个评价详情
- 批量更新回复状态
- 缓存测试

### 3. AI生成回复 (ai-reply.test.ts)
- AI回复生成（多种风格）
- 自定义回复指令
- 差评原因分类
- 质量问题识别
- 物流问题识别
- 服务问题识别
- 期望差距识别
- 使用限制检查
- 回复质量评估

### 4. 订阅购买 (subscription.test.ts)
- 套餐列表获取
- 套餐信息验证
- 套餐缓存
- 用户订阅信息
- 每日使用限制
- 订阅升级
- 订单管理
- 订阅取消
- 使用量统计

## 运行测试

### 快速开始
```bash
cd smart-customer-service-platform

# 确保开发服务器运行
npm run dev

# 运行所有API测试
node __tests__/e2e/simple-runner.js

# 运行指定模块
node __tests__/e2e/simple-runner.js --module=auth
```

### 使用Shell脚本
```bash
chmod +x run-e2e-tests.sh
./run-e2e-tests.sh --all
```

### Playwright UI测试
```bash
npm install -D @playwright/test
npx playwright test
```

## 关键设计决策

1. **独立脚本优先** - 提供不依赖测试框架的简单运行方式
2. **动态测试数据** - 使用时间戳生成唯一手机号避免冲突
3. **多层次测试** - API测试 + UI测试双重覆盖
4. **容错设计** - 测试包含跳过逻辑，适应不同环境
