# 执行计划

## 概述

加载计划，批判性审查，逐步执行所有任务，完成后报告。

**开始时声明：** "我正在使用 executing-plans 技能来执行这个计划。"

## 流程

### 第一步：加载并审查计划

1. 读取计划文件
2. 批判性审查 — 识别任何疑问或顾虑
3. 如果有顾虑：在开始前向用户提出
4. 如果没有顾虑：创建 TodoWrite 任务列表并开始

### 第二步：执行任务

对每个任务：

1. 标记为 `in_progress`
2. 严格按步骤执行（计划有细粒度步骤）
3. 按指定方式运行验证
4. 标记为 `completed`
5. 更新 `/home/z/my-project/worklog.md`

### 第三步：完成开发

所有任务完成并验证后：

- 宣布：所有任务执行完毕
- 运行完整测试套件
- 总结完成情况

## 何时停止并求助

**立即停止执行：**

- 遇到阻塞（缺少依赖、测试失败、指令不清晰）
- 计划有严重缺口导致无法开始
- 你不理解某个指令
- 验证反复失败

**要求澄清而不是猜测。**

## 铁律

1. **先审查计划，再动手**
2. **严格按步骤执行**，不跳步
3. **不跳过验证**
4. **遇到阻塞时停止，不猜测**
5. **每完成一个任务就更新 worklog.md**

## 与 worklog 配合

每个任务完成后，在 worklog.md 追加：

```markdown
---
Task ID: <task-id>
Agent: <agent-name>
Task: <task description>

Work Log:
- <具体步骤1>
- <具体步骤2>

Stage Summary:
- <关键结果>
```

## 执行追踪

使用 TodoWrite 实时追踪进度，确保：
- 同一时间只有一个任务处于 `in_progress`
- 完成后立即标记为 `completed`
- 所有任务完成后做最终验证
