# 智能客服平台（淘宝版）设计文档

**日期：** 2026-04-10
**状态：** 已批准

## 目标
构建一个AI驱动的淘宝智能客服工作台，涵盖差评处理、客服回复、实时聊天、订单管理四大核心场景。

## 功能模块

### 1. 数据看板（Dashboard）
- 评价概览：今日新评价数、差评数、回复率、AI处理率
- 评价趋势图（7天/30天）
- 差评原因分类饼图（物流/质量/服务/描述不符/其他）
- 实时聊天会话数、待处理工单数

### 2. 评价管理
- 全部评价列表（支持按星级、情感标签、时间筛选）
- AI情感标注（正面/负面/中性）
- 差评详情：商品信息、买家信息、评价内容、AI分析结果
- AI自动生成2-3个回复方案，支持一键采纳/编辑后发送
- 回复状态追踪（待回复/已回复/AI已处理）

### 3. AI智能回复
- 基于GLM大模型，根据评价内容+商品信息+历史回复生成回复
- 回复风格可选（诚恳道歉型/专业解释型/补偿方案型）
- 回复质量评分

### 4. 实时聊天
- 模拟旺旺聊天界面
- AI自动应答买家消息
- 常见问题快捷回复
- 会话历史记录

### 5. 订单管理
- 订单列表（待发货/已发货/已完成/退款中）
- 物流状态展示（模拟数据）
- 按订单筛选关联评价
- 订单详情页

### 6. 回复模板库
- 预设模板分类（物流问题/质量问题/服务态度/描述不符）
- 自定义模板CRUD
- AI生成新模板

### 7. 数据报表
- 差评原因分析柱状图
- 回复效果追踪（回复后追评好评率）
- AI处理效率统计
- 客服KPI仪表盘

### 8. 店铺管理
- 多店铺切换
- API授权配置页面（模拟）

## 技术架构
- Next.js 16 App Router + TypeScript
- Tailwind CSS 4 + shadcn/ui
- Prisma + SQLite
- z-ai-web-dev-sdk (GLM) AI回复生成
- Socket.io 实时聊天
- Recharts 数据可视化

## 数据模型
- Shop（店铺）
- Review（评价）
- ReviewReply（评价回复）
- Order（订单）
- ChatSession（聊天会话）
- ChatMessage（聊天消息）
- ReplyTemplate（回复模板）

## UI设计
- 深色侧边栏导航 + 白色内容区
- 专业商务风格
- 响应式布局（移动端适配）
- 颜色主题：橙色系（呼应淘宝品牌色）

## 非目标
- 不做真实的淘宝API对接（用模拟数据）
- 不做支付功能
- 不做多语言支持
- 不做用户注册/登录（单用户演示）

## 风险与缓解
| 风险 | 缓解措施 |
|------|---------|
| AI回复质量不佳 | 提供多方案选择+人工编辑 |
| 数据量过大 | 分页+懒加载 |
| 实时聊天延迟 | Socket.io长连接 |
